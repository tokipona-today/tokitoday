UV is a rust‚Äëpowered Python package & project manager that handles Python versions, virtual environments, dependencies, lockfiles, scripts and tools‚Äîall insanely fast and unified.

üöÄ What is uv?

uv (pronounced ‚Äúyou‚Äëvee‚Äù) is a single tool that replaces pip, pipx, pyenv, pip-tools, venv, even poetry.
It supports cross-platform lockfiles, Python version management, script execution with embedded metadata, and legacy pip interface compatibility‚Äîwritten in Rust for speed and reliability.

üåü Key Features

  • Python version manager: install and use specific versions or PyPy builds, entirely managed by uv. It auto-downloads missing versions as needed.
  • Automatic virtual environments: when you run uv run or uv sync in a project, uv will auto-create a .venv folder if none exists‚Äîno prompt needed.
  • Dependency management: uv add, uv remove, uv sync, uv lock vs. pip compile, pip sync, etc.‚Äîa universal lockfile (uv.lock) ensures consistent installs across platforms.
  • Script execution: run single-file scripts with inline dependency declarations and optional reproducibility via exclude-newer timestamps.
  • Pip-compatible CLI: uv pip install, uv pip freeze, uv pip compile, etc., work just like familiar pip tools‚Äîwith massive performance gains.
  • Tool runner: commands like uvx (alias for uv tool run) let you run tools like ruff or pycowsay in isolated environments.

üí° Tips & Smart Tricks

  • uv python install 3.10 3.11 3.12 ‚Äî install multiple Python versions
  • uv python pin 3.11 ‚Äî write .python-version to pin interpreter

üö´ Limitations & Notes

  • .venv is created automatically ‚Äî but not .env (for environment variables)
  • No automatic env var loading: use direnv, dotenv, etc.
  • Python binaries are not auto-symlinked ‚Äî use uv python pin or link manually

üß† Why switch?

  • Speed: 10√ó to 100√ó faster than pip/poetry/pip-tools
